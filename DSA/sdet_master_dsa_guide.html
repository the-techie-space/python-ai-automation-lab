The 8 master patterns — recognise these instantly

Every DSA problem maps to one of these. Train your brain to ask "which pattern is this?" first.

Pattern 1: Two Pointers

When to use: Sorted array, find pair/triplet, check palindrome, remove duplicates

Signal words: "sorted array", "pair that sums to", "reverse", "palindrome"

left, right = 0, len(arr)-1
while left < right:
    if condition: return result
    elif too_small: left += 1
    else: right -= 1

Your solved problems: Two Sum II, Valid Palindrome, 3-Sum, Container with Most Water

Pattern 2: Sliding Window

When to use: Subarray/substring problems, contiguous elements, find max/min/longest/shortest

Signal words: "subarray", "substring", "window", "contiguous", "longest", "shortest"

# Fixed window
for i in range(len(arr)):
    window.add(arr[i])
    if i >= k: window.remove(arr[i-k])

# Variable window
left = 0
for right in range(len(arr)):
    # expand window
    while invalid_condition:
        # shrink window
        left += 1
    # record result

Your solved problems: Longest Substring No Repeat, Min Window Substring, Find Anagrams

Pattern 3: HashMap / HashSet

When to use: Frequency count, existence check, grouping, O(1) lookup needed

Signal words: "count", "frequency", "group", "find duplicate", "two sum"

freq = {}
for x in arr:
    freq[x] = freq.get(x, 0) + 1
# or: seen = set() for existence only

Your solved problems: Two Sum, Valid Anagram, Group Anagrams, LRU Cache, Subarray Sum=K

Pattern 4: Fast + Slow Pointers

When to use: Linked list cycle, find middle, kth from end

Signal words: "cycle", "middle of list", "loop", "meets"

slow = fast = head
while fast and fast.next:
    slow = slow.next
    fast = fast.next.next
    if slow == fast: return True  # cycle!

Pattern 5: Binary Search

When to use: Sorted array, find target, search space can be halved

Signal words: "sorted", "find minimum", "search", "rotated array"

left, right = 0, len(arr)-1
while left <= right:
    mid = (left + right) // 2
    if arr[mid] == target: return mid
    elif arr[mid] < target: left = mid + 1
    else: right = mid - 1

Your solved problems: Binary Search, Search Rotated Array, Find Min in Rotated

Pattern 6: Stack / Monotonic Stack

When to use: Next greater/smaller element, valid parentheses, temperature problems

Signal words: "next greater", "valid brackets", "temperature", "buildings"

stack = []
for i, val in enumerate(arr):
    while stack and arr[stack[-1]] < val:
        idx = stack.pop()
        result[idx] = val   # next greater found!
    stack.append(i)

Pattern 7: Tree DFS / BFS

When to use: Tree/graph traversal, level order, path sum, height

Signal words: "tree", "level", "path", "height", "depth", "ancestor"

# DFS — recursive
def dfs(node):
    if not node: return 0
    left = dfs(node.left)
    right = dfs(node.right)
    return 1 + max(left, right)

# BFS — level order
from collections import deque
queue = deque([root])
while queue:
    node = queue.popleft()
    if node.left: queue.append(node.left)
    if node.right: queue.append(node.right)

Pattern 8: Dynamic Programming

When to use: Optimisation, count ways, overlapping subproblems

Signal words: "max/min", "count ways", "can you reach", "longest subsequence"

# Memoization top-down
from functools import lru_cache
@lru_cache(maxsize=None)
def dp(i): return dp(i-1) + dp(i-2)

# Tabulation bottom-up
dp = [0] * (n+1)
dp[0], dp[1] = 0, 1
for i in range(2, n+1):
    dp[i] = dp[i-1] + dp[i-2]

Python one-liners interviewers love to see

# Reverse a list / string
arr[::-1]
s[::-1]

# Sort with key
arr.sort(key=lambda x: x[1])  # sort by second element
sorted(words, key=len)         # sort by length

# Flatten nested list
flat = [x for row in matrix for x in row]

# Counter — most common
from collections import Counter
Counter(arr).most_common(3)    # top 3 frequent

# DefaultDict
from collections import defaultdict
d = defaultdict(list)          # default is empty list

# Deque — O(1) both ends
from collections import deque
q = deque()
q.appendleft(x)  # push left O(1)
q.popleft()      # pop left  O(1)

# Heap (min-heap)
import heapq
heapq.heappush(heap, val)
heapq.heappop(heap)            # always pops smallest

# Max-heap trick (negate values)
heapq.heappush(heap, -val)

# Prefix sum
prefix = [0] * (len(arr)+1)
for i, v in enumerate(arr):
    prefix[i+1] = prefix[i] + v
# sum(arr[i:j]) = prefix[j] - prefix[i]

# Binary search built-in
import bisect
bisect.bisect_left(arr, target)  # insertion point

SDET-specific: always say these in interviews

# 1. Clarify constraints first
"What is the size of input? Can it be empty? Are there duplicates?"

# 2. State your approach before coding
"I'll use a sliding window here because we need a contiguous subarray."

# 3. Always mention complexity
"This runs in O(n) time and O(n) space due to the HashMap."

# 4. Mention edge cases proactively
"I'll handle empty input, single element, and all-same-element cases."

# 5. Write test cases (SDET superpower)
assert solution([]) == 0          # empty
assert solution([1]) == 1         # single element
assert solution([1,1,1]) == 1     # all same

5-minute interview warm-up — read this before every interview

The SDET interview formula — say this structure every time

Step 1 — UNDERSTAND  (2 min)
  "Let me restate the problem. We have X and need to return Y."
  "Can input be empty? Negative numbers? Duplicates?"

Step 2 — PATTERN     (1 min)
  "I see [signal word] so this is a [pattern] problem."
  "I'll use [data structure] because [reason]."

Step 3 — APPROACH    (2 min)
  Explain brute force first, then optimal.
  "Brute force is O(n²). I can do better with..."

Step 4 — CODE        (15 min)
  Talk while you code. Say what each line does.

Step 5 — TEST        (5 min)  ← YOUR SDET SUPERPOWER
  "Let me test with: normal case, empty input, single element, edge case."
  Write assert statements. Dry run on example.

Step 6 — COMPLEXITY  (1 min)
  "Time: O(n) because... Space: O(n) because..."

Key things learned from our sessions together

Two Pointers trick

Sort first if not sorted. Move left when sum too small, right when too large. For palindrome: skip non-alphanumeric.

Sliding Window rule

Expand with right pointer. Shrink from left when window is invalid. Record result AFTER while loop exits.

HashMap golden rule

Always use .get(key, 0) not direct access. Store complement AFTER checking. Delete zero keys to keep dict clean.

formed == required

In Min Window Substring, only record/shrink when window is valid. This is the hardest pattern — you cracked it!

Linked list survival

Never lose head pointer. curr.next = curr.next.next skips a node. Use dummy head to simplify edge cases.

Binary search rule

left <= right for exact search. left < right for range search. mid = left + (right-left)//2 avoids overflow.
